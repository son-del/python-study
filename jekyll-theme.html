Jekyll 테마란?

GitHub Pages에서 블로그 디자인을 쉽게 바꾸는 방법

1. Jekyll이란?

Jekyll은 정적 사이트 생성기(Static Site Generator)예요. Markdown 파일이나 HTML 템플릿을 이용해 자동으로 웹사이트를 만들어줍니다. GitHub Pages는 내부적으로 Jekyll을 지원해서, 별도 설치 없이 블로그처럼 꾸밀 수 있습니다.

2. Jekyll 테마란?

테마(theme)는 Jekyll 사이트의 디자인과 구조를 미리 정의한 패키지예요. CSS, 레이아웃, 글 목록, 내비게이션 등을 포함하고 있어서, 설정 한 줄만 추가해도 사이트의 전체 스타일을 바꿀 수 있습니다.

  • 공식 테마: minima, jekyll-theme-cayman, jekyll-theme-slate 등
  • 사용자 제작 테마도 Jekyll 공식 사이트에서 설치 가능

3. 테마 적용 예시

_config.yml 파일은 Jekyll 사이트의 설정 파일이에요. 여기에 아래처럼 테마를 지정하면 됩니다.

# _config.yml
title: "파이썬 공부 블로그"
description: "GitHub Pages와 Jekyll로 만드는 정적 블로그"
theme: jekyll-theme-cayman
markdown: kramdown

위처럼 theme: 값을 바꾸면, 사이트 전체가 선택한 테마의 색상과 스타일로 자동 변합니다.

4. GitHub Pages에서 적용하기

GitHub Pages는 Jekyll을 내장하고 있어서 별도의 설치 과정이 없습니다. 저장소 루트(예: python-study)에 _config.yml 파일을 추가하고 테마 이름만 지정하면 됩니다.

#입문 #자료형 #제어문 #함수 #문제풀이 
# _config.yml 예시
theme: jekyll-theme-minimal

이후 저장소를 커밋하고 배포하면, 페이지가 지정한 테마로 새롭게 렌더링됩니다.

5. 직접 커스터마이징하기

Jekyll 테마를 적용한 뒤에도 assets 폴더나 _layouts 디렉터리를 만들어 일부 스타일을 덮어쓸 수 있습니다. 

# 예: CSS 수정
assets/css/style.scss

---
---

@import "{{ site.theme }}";

body {
  font-family: 'Noto Sans KR', sans-serif;
  background-color: #fefcf9;
}

6. 결과 미리보기

GitHub Pages 설정에서 Branch: main, Folder: /(root)로 지정한 후 https://사용자이름.github.io/저장소이름/ 로 들어가면 테마가 적용된 사이트를 볼 수 있습니다.
